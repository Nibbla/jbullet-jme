Contruye un PhysicsNode con el modelo indicado, usando el tipo de geometría de colisión especificado.

Parameters:
modelo - el modelo que se usará para representar el nodo de físicas
collisionClass - el tipo de geometría de colisión que se usará para calcular las colisiones del modelo
Returns:
dependiendo de la clase de colisión indicada, se deolverán los siguientes tipos de objetos:
  • GImpactCollisionShape, MeshCollisionShape: Se devuelve un objeto de tipo Node que contiene tantos objetos de tipo PhysicsNode como submayas (TriMesh) que contenga el modelo indicado. Es decir, se crea u n nodo físico por cada submaya.
  • CompoundCollisionShape: No permitido en este método. Usar el método que admite 3 parámetros para configurar el tipo de forma que compondrán la maya resultante.
  • Resto de tipos de colisión: Retorna un objeto de tipo PhysicsNode
una instancia de PhysicsNode con el modelo asociado.

compoundChildShapeClass - el tipo de CollisionShape que se utilizará para cada una se las submayas cuando se quiere crear un nodo con una CompoundCollisionShape. Es obligatorio cuando se indica como collisionClass una clase del tipo CompoundCollisionShape. Para el resto de tipos de formas de colisión, este parámetro se ignora.
Returns:
dependiendo de la clase de colisión indicada, se deolverán los siguientes tipos de objetos:

  • CompoundCollisionShape: Se devuelve un objeto de tipo PhysicsNode que contiene una geometría de colisión compuesta por las geometrías del modelo indicado. La CompoundCollisionShape estará formadas por geometrias de colisión del tipo especificado por el parámetro compoundChildShapeClass
